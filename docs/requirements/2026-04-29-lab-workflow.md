## Основные сущности и термины

- **Проба**: исходная физическая единица (пробирка/материал) с полями описания, источника и хранения.
- **ДНК-экстракт**: результат этапа выделения ДНК для пробы (обычно один раз на пробу, но с учётом возможного «повторного выделения» как связи/истории).
- **Маркер / фрагмент**: участок, который амплифицируется (потенциально множество маркеров на одну пробу).
- **Задание**: набор работ, сформированный на основании параметров (например, комбинация праймеров + концентрация матрицы + общий объём), с возможностью назначения исполнителя и приоритета.
- **Операция этапа**: запись «кто/когда/где/как» + артефакты (файлы, гель-картинки) + результат (статусы ok/weak/double/fail).

## Этапы процесса (пайплайн)

### 1) Подготовка пробы (до лабораторной работы)

- **заполнение карточки/строки пробы**: таксон/название, комментарий, где собран материал, кем собран, номер сбора, где хранится, гербарий и гербарный номер, связи/`Connections` (например, повторное выделение, несколько анализов).

### 2) Выделение ДНК

- **массовый выбор проб** (галочки) для постановки работы «выделение ДНК».
- **массовое применение параметров** к выбранным пробам:
  - дата
  - лаборатория (выбор из ограниченного списка)
  - метод (выбор из ограниченного списка)
  - оператор/исполнитель (обычно текущий пользователь)
- **формирование отчёта/бланка для печати** по партии работ (для подшивки в бумажный журнал).

#### Статусы для выделения ДНК

- **не выделена**
- **выделена**
- **материал/экстракт закончился** (экстракт физически отсутствует, «пробирку выкинули/пустая»)

### 3) Измерение концентрации ДНК (опционально)

- **массовое применение**: дата, оператор, место.
- **внесение результатов**:
  - численные значения (если есть прибор)
  - или пропуск этапа (решение «не измерять»)
- **привязка файлов/артефактов**:
  - картинка/скрин/файл прибора (может быть общий файл на партию или отдельный на каждую пробу)
  - комментарии

### 4) Амплификация (основной «многократный» этап)

Особенности:
- на одну пробу может быть **несколько маркеров/фрагментов**
- по каждому маркеру может быть **несколько попыток** (в разные даты, разными людьми, в разных лабораториях, с разными параметрами)

#### Параметры задания/попытки амплификации

- прямой праймер
- обратный праймер
- количество ДНК-матрицы (концентрация/объём матрицы — как отдельный параметр)
- общий объём реакции
- (опционально) примечание/вариативный реагент
- приоритет (ранг важности)
- исполнитель (конкретный пользователь или «всем»)

#### Механика «заданий»

- **формирование заданий** (work queue) до выполнения работ
- **группировка/сегментация** задания по одинаковым параметрам (праймеры + матрица + общий объём), потому что физически работы выполняются партиями
- **генерация печатного отчёта** на выбранный поднабор (например, 24 из 100)
- **взятие в работу**:
  - выбранные строки удаляются из «задания»
  - вместо этого фиксируется «операция» в истории (кто/когда/где/параметры)
- **результат** как «гель-картинка» + классификация:
  - `ok` (полоса хорошая, можно переходить дальше)
  - `weak` (полоса слабая, иногда можно попытаться, чаще — переделка)
  - `double` (две полосы, обычно требуется переделка/смена параметров)
  - `fail`/«не получилось»
- **возврат в задание**: если нужно переделать — создаётся новое задание с обновлёнными параметрами; история предыдущих попыток сохраняется.

### 5) Очистка амплификата

- входной фильтр: фрагменты со статусом `ok` (и часть `weak`, если принято решение пробовать)
- **взятие партии в работу** (8/16/96 — партийность по типовым наборам)
- фиксация: дата, исполнитель, методика
- **внесение концентрации после очистки** (удобный ввод в рамках выборки/партии, чтобы снизить ошибки сопоставления)
- поддержка **качественной оценки** (гель «годно/не годно») и/или количественной концентрации

### 6) Секвенирование (аутсорс)

- фильтр «готово к секвенированию» (часто на основе концентрации/качества)
- выбор партии, фиксация:
  - куда отправлено (лаборатория)
  - дата отправки
  - кто отправил (опционально)
  - отметка варианта «очистка на стороне лаборатории» (если этап 5 пропускается локально)
- внесение результата по партии после получения данных:
  - `success` / `fail` / `retry`
  - причина `fail` (опционально): «получился другой организм», «не читается из-за смеси», «слишком слабая концентрация» и т. п.
- сохранение результатов:
  - **номер GenBank** (для маркера/фрагмента)
  - **сырой файл** (1–2 файла: «сырой» и «обработанный» — обработанный необязателен)
  - **текст последовательности** (буквы), чтобы хранить не только номер

## Роли и права

- роли: **админ / редактор / читатель**
- постановка и редактирование заданий: **админ и редактор**
- чтение истории/результатов: минимум читатель

## Требования к фильтрам и представлениям

- выбор **этапа** определяет:
  - набор видимых полей
  - доступные фильтры
  - доступные массовые действия
- ключевые фильтры:
  - «ДНК не выделена» (для этапа выделения)
  - «готово к очистке» (ok + часть weak по решению)
  - «готово к секвенированию» (на основе концентрации/качества/статуса)
  - по заданию: маркер, комбинация праймеров, матрица, общий объём, приоритет, исполнитель
- сортировка по концентрации для принятия решения об отправке (с учётом разных порогов в лабораториях)

## Отчёты и печать

- генерация печатного «бланка»/отчёта для каждой партии работ (выделение, амплификация-подзадачи, очистка, отправка на секвенирование)
- отчёт должен включать дату/лабораторию/метод/исполнителя/параметры и список номеров

## Файлы и артефакты

- прикрепление файлов к операциям этапов:
  - измерения концентрации (картинки/файлы прибора)
  - амплификация (гель-картинки)
  - секвенирование (сырой файл, опционально обработанный файл)
- поддержка «один файл на партию» и «файл на запись»

## Импорт, экспорт, резервирование

- обязательное наличие **экспорта** в плоские таблицы (CSV/аналог Excel) для долговременной сохранности и миграций
- импорт существующих Excel-данных:
  - автоматическое сопоставление максимально возможного количества полей
  - выделение «проблемных» мест (неоднородные даты/смешанные поля вроде «Давыдов 5295»)
  - интерактивное уточнение маппинга на этапе импорта (пакетные решения там, где возможно)
- редактирование «старых» данных — по необходимости и точечно; основной фокус — корректное ведение «с момента внедрения системы».

## Мобильное использование и идентификаторы

- использование интерфейса на планшете/мобильно (в лаборатории)
- опционально: быстрый переход к образцу по идентификатору (QR/штрих‑код), если в дальнейшем появятся устойчивые метки на пробирках/коробках.
