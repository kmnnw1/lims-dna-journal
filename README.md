## Описание

Многоуровневое web-приложение для системного учёта лабораторных проб, контроля этапов выделения ДНК и ведения ПЦР‑журнала. Структура данных, процедура работы с образцами, разграничение ролей, импорт и экспорт данных пригодны для лабораторных исследовательских и внутренних инфраструктур, рассчитаны на долговременное хранение и сопровождение.

Посредством современных технологий приложение обеспечивает сохранность данных, прозрачность процессов, удобство интеграции и минимальный порог вхождения для новых пользователей. Документация ориентирована на специалистов любого уровня — вход не требует узкопрофильной подготовки.

## Развёртывание

### Один шаг после клонирования (предустановлен Node.js 20+)

```bash
node scripts/install-all.mjs
```

Данный скрипт выполнит инициализацию зависимостей, создание и заполнение `.env` на основе примера, генерацию случайного NEXTAUTH_SECRET (при отсутствии), запуск Prisma генерации и push схемы, сборку и запуск приложения. Для остановки используйте сочетание клавиш `Ctrl+C`.

Эквивалентные ручные действия:
```bash
npm install
npm run setup
npm start
```

Для Windows предусмотрен `scripts/install.ps1`; Linux/macOS — `scripts/install.sh` (необходим chmod +x).

### Docker

```bash
cp .env.example .env
# В .env укажите значения NEXTAUTH_SECRET и (опционально) NEXTAUTH_URL
mkdir -p lab-data
docker compose up --build
```

База данных размещается внутри тома `./lab-data` (каталог игнорируется в git). DATA_XLSX_PATH указывает путь к исходному файлу импорта Excel внутри контейнера.

### Контроль работы

Эндпоинт HTTP `GET /api/health` возвращает JSON с ключом `ok`, версией приложения и меткой времени. Предназначено для сценариев мониторинга.

## Быстрый старт

1. Node.js 20 или новее ([nodejs.org](https://nodejs.org/en/download)).
2. Клонируйте репозиторий и выполните установку зависимостей:
    ```bash
    git clone <url-репозитория> lab-journal
    cd lab-journal
    npm install
    ```
3. Настройте переменные окружения:
    ```bash
    cp .env.example .env
    ```
    Минимальная конфигурация `.env`:
    ```env
    DATABASE_URL="file:./dev.db"
    NEXTAUTH_SECRET="<openssl rand -base64 32>"
    NEXTAUTH_URL="http://localhost:3000"
    ```
    При необходимости укажите DATA_XLSX_PATH для задания своего пути к файлу импорта.
4. Первичная инициализация базы и запуск сервера для разработки:
    ```bash
    npx prisma generate
    npx prisma db push
    npm run dev
    ```

    Интерфейс доступен на http://localhost:3000.  
    Если таблица пользователей пуста, использовать логин/пароль admin/admin для инициализации первой учетной записи администратора.

## Импорт данных

Исходный файл `data.xlsx` размещается в корне проекта. Импорт производится через административную панель приложения. DATA_XLSX_PATH может быть изменен в настройках окружения.

---

## Подготовка production-сборки

```bash
npm run build
npm run start
```

Порт сервера определяется переменной окружения `PORT` (по умолчанию 3000). Для работы во внешней среде укажите корректный NEXTAUTH_URL. Для перехода на PostgreSQL измените параметры DATABASE_URL и провайдера в схемe Prisma, затем выполните миграцию.

---

## Резервное копирование

```bash
npm run backup
```

Создаётся копия файла базы данных `prisma/dev.db` с отметкой времени в каталоге `backups/`. Каталог защищён от попадания в git.

---

## Работа при потере сети и параллельное редактирование

Вся клиентская активность выполняется через API на хосте сервера. Если соединение потеряно, изменения не сохраняются до восстановления связи; отдельной очереди на синхронизацию не предусмотрено. При параллельной работе над одной записью фиксируется последнее успешное изменение; строгий контроль консистентности версий не реализован.
