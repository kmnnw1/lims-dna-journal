# 🧬 LIMS | Журнал проб ДНК и ПЦР

[![TypeScript](https://img.shields.io/badge/TypeScript-5-3178C6?logo=typescript&logoColor=white)](https://www.typescriptlang.org/)
[![Prisma](https://img.shields.io/badge/Prisma-ORM-2D3748?logo=prisma&logoColor=white)](https://www.prisma.io/docs)
[![Material Design 3](https://img.shields.io/badge/UI-Material_Design_3-ea4335?logo=materialdesign&logoColor=white)](https://m3.material.io/)

Современная система управления лабораторной информацией (LIMS) для системного учёта биологических проб, контроля этапов выделения ДНК и ведения ПЦР‑журнала. 

Проект разработан с фокусом на производительность, надежность данных и **премиальный пользовательский опыт (UX/UI)** на базе концепции *Material Design 3 Expressive*.

---

## ✨ Ключевые особенности

* **🎨 Material Design 3 Expressive**: Мягкие формы, выразительная типографика (Roboto Flex), тональные поверхности и плавная поддержка темной/светлой темы.
* **📱 Progressive Web App (PWA)**: Приложение устанавливается на телефон или ПК, поддерживает кэширование статики и имеет красивую офлайн-заглушку при потере сети.
* **🚀 Высокая производительность**: Построено на Next.js 16 (App Router) с оптимизированным рендерингом и быстрыми переходами.
* **🔐 Ролевая модель (RBAC)**: 
  * `ADMIN` — управление пользователями, импорт Excel, полная очистка БД.
  * `EDITOR` — добавление и редактирование проб, занесение результатов ПЦР.
  * `READER` — только безопасный просмотр данных.
* **📊 Умный импорт/экспорт**: Автоматический парсинг сложных Excel-файлов (`data.xlsx`) и выгрузка текущего среза базы в CSV.

---

## 🛠 Технологический стек

* **Фронтенд**: Next.js 16, React 19, Tailwind CSS v4, Lucide Icons.
* **Бэкенд**: Next.js API Routes, NextAuth.js (JWT-сессии).
* **База данных**: Prisma ORM, SQLite (с легким переходом на PostgreSQL).
* **Тестирование**: Playwright (E2E тесты адаптированы под MD3), Vitest.

## 🚀 Быстрый старт (Локально)

Убедитесь, что у вас установлен **Node.js 20 или новее**.

1. Клонируйте репозиторий:
   ```bash
   git clone <url-репозитория> lab-journal
   cd lab-journal
   ```

2. Запустите автоматический скрипт установки:
   ```bash
   # Для Windows, Linux и macOS:
   node scripts/install-all.mjs
   ```
   *Скрипт автоматически установит зависимости, создаст `.env` с криптографическим ключом, сгенерирует Prisma Client, накатит схему БД и запустит сервер на `http://localhost:3000`.*

3. **Первый вход**: Если база пуста, используйте логин `admin` и пароль `admin`. Система автоматически создаст учетную запись главного администратора.

## 🐳 Развёртывание (Docker)

Для production-среды рекомендуется использовать Docker. База данных и файлы импорта хранятся в изолированном томе `./lab-data`.

1. Скопируйте конфигурацию окружения:
   ```bash
   cp .env.example .env
   ```
2. Сгенерируйте `NEXTAUTH_SECRET` (например, командой `openssl rand -base64 32`) и впишите его в `.env`. Укажите актуальный `NEXTAUTH_URL` (например, `https://lims.my-lab.com`).
3. Запустите контейнеры:
   ```bash
   docker compose up -d --build
   ```

## 🗃 Работа с данными

### Импорт из Excel
Поместите ваш файл `data.xlsx` в корень проекта (или в папку `lab-data/` при использовании Docker). Зайдите в систему под учетной записью `ADMIN`, перейдите в раздел **Администрирование** и нажмите «Импортировать». Текущая база будет очищена, а данные из всех листов Excel — бережно перенесены в систему.

### Резервное копирование БД
Вы можете создать горячую резервную копию базы данных SQLite (сохраняется в папку `backups/`):
```bash
npm run backup
```

## ⌨️ Горячие клавиши и навигация

Система поддерживает управление без мыши для ускорения рутинных операций (особенно полезно при работе в перчатках в лаборатории):
* `Ctrl + K` (или `⌘ + K`) — Глобальная палитра команд (поиск проб и быстрые действия).
* `/` — Фокус на главном поле поиска.
* `N` — Добавить новую пробу (только для EDITOR/ADMIN).
* `?` — Показать справку по горячим клавишам.
