# LIMS | DNA Lab Journal

Система управления лабораторной информацией для автоматизации учета биологических проб, экстракции ДНК и мониторинга процессов ПЦР.

## Технологический стек

| Слой | Технология | Назначение |
| :--- | :--- | :--- |
| **Framework** | Next.js 16.2 (App Router, Turbopack) | Основной движок приложения |
| **Logic** | React 19, TypeScript | Клиентская логика и типизация |
| **Styling** | Tailwind CSS v4, MD3 Expressive | Современный интерфейс и токены MD3 |
| **Database** | Prisma ORM, SQLite | Хранение данных (локально) |
| **Auth** | NextAuth.js | Одноразовые токены (Hiddify-style) |
| **Testing** | Playwright | Сквозное тестирование (E2E) |

## Функциональные возможности

| Область | Описание |
| :--- | :--- |
| **Реестр проб** | Учет таксономических данных, координат сбора, концентрации ДНК и статусов качества. |
| **ПЦР-журнал** | История попыток амплификации с привязкой к маркерам (ITS и др.) и используемым праймерам. |
| **Безопасность** | Беспарольный вход по одноразовым токенам для исключения брутфорс-атак. |
| **Администрирование** | Импорт из Excel (data.xlsx), экспорт таблиц в CSV, управление ролями участников. |
| **UI/UX** | Оптимистичные обновления, MD3 State Layers, нативная анимация интерфейса. |

## Развертывание и запуск

### Зависимости
- Node.js >= 20.0.0
- Установленные зависимости (`npm install`)

### Локальная установка (Manual)
1. Настройте файл окружения: `cp .env.example .env`
2. Инициализируйте базу и соберите клиент: `npm run setup`
3. Запустите в режиме разработки: `npm run dev`

### Контейнеризация (Docker)
Система поддерживает запуск через Docker Compose (рекомендуется для Linux-серверов):
```bash
docker compose up --build -d
```
Том базы данных монтируется в `./lab-data`.

## Командный интерфейс (NPM)

| Команда | Описание |
| :--- | :--- |
| `npm run dev` | Запуск сервера разработки на [http://localhost:3000](http://localhost:3000) |
| `npm run setup` | Автоматическая подготовка БД и сборка проекта |
| `npm run auth` | Генерация одноразовой ссылки для входа в систему |
| `npm run build` | Компиляция production-сборки проекта |
| `npm run start` | Запуск приложения из собранного билда |
| `npm run backup` | Создание резервной копии базы данных SQLite |

## Структура проекта

- `app/`: Маршрутизация (UI) и API-эндпоинты.
- `components/`: Реализация MD3-компонентов и модальных окон.
- `lib/`: Ядро системы (Auth, Database, Excel Parser).
- `prisma/`: Определение схемы данных и миграции.
- `scripts/`: Служебные инструменты (автоматизация, бэкапы).
